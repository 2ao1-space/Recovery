# How-Web-Works

#### 1- DNS Server

- لما المستخدم بيبحث عن ال github.com المتصفح بيبعت لسيرفر الDNS و ف الDNS مش بيفهم كلام بيفهم ارقام ف السيرفر بيطلب الIP
  بتاع الموقع.

#### 2- connection between server and client

- و دا بيحصل ف 3 خطوات
  1- الclient بيطلب التواصل مع ال server (SYN).
  2- الserver بيرد تمام انا متاح (SYN & ACK).
  3- الserver بيبعت رد لل client (ACK) انه تمام ممكنن نبدا نقل البيانات و ان النقل امان.

#### 3- http stage

- لما دخلنا مرحله نقل البيانات ال client بيبعت طلب لل server ان انا محتاج الموقع (github.com) و بيبعت مع الطلب الheaders زي نوع المتصفح و اللغه و هكذا.
- الserver لما بيلاقي الموقع موجود ف بيرد ب (200 انه تمام) + صفحة ال html.
- الclient بياخد الصفحه و يعرضها و بيحولها لtree في الميموري اسمها ال DOM و كل tag هو عقد فDOM و منها الjs بيقدر يرتب الصفحه
- الclient يلاقي في الhtml كود css وال js ف ال client بيبعت طلبات تانيه لتحميل باقي الملفات الcss الاول و بيكون الCSSOM و دا بيوقف كل حاجة و عشان كدة بنحط الjs اخر ال body
- الbrowser بيدمج الDOM & CSSOM و بيعمل Render tree و بعدين يحسب مكان كل عنصر و بيرسم كل حاجه علي الشاشهو ده اللي بيشوفه المستخدم في الاخر
