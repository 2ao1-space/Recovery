# How-Web-Works

#### 1- DNS Server

- لما المستخدم بيبحث عن ال github.com المتصفح بيبعت لسيرفر الDNS و DNS يحوّل اسم الدومين إلى IP address.

#### 2- connection between server and client

- و دا بيحصل ف 3 خطوات
  1- الclient بيطلب التواصل مع ال server (SYN).
  2- الserver بيرد تمام انا متاح (SYN & ACK).
  3- الserver بيبعت رد لل client (ACK) انه تمام ممكنن نبدا نقل البيانات و يتم إنشاء اتصال موثوق لنقل البيانات، أما التشفير فيتم لاحقًا عن طريق TLS لو الموقع HTTPS.

#### 3- http stage

- لما دخلنا مرحله نقل البيانات ال client بيبعت طلب لل server ان انا محتاج الموقع (github.com) و بيبعت مع الطلب الheaders زي نوع المتصفح و اللغه و هكذا.
- المتصفح يعرف إنه HTTPS من أول ما شاف الـ URL نفسه و لو https هل عنده شهادة SSL/TLS صالح ولا لا لو تمام المتصفح بيحط علامه قفل و يعتبر الاتصال امن و بيتم استخدام نظام الTLS عشان يشفر الداتا بحيث ان محدش يعترض البيانات بتاعت الموقع
- الserver لما بيلاقي الموقع موجود ف بيرد ب (200 انه تمام) + صفحة ال html.
- الclient بياخد الصفحه و يعرضها و بيحولها لtree في الميموري اسمها ال DOM و كل tag هو عقد فDOM ومنها Js يقدر يتعامل مع عناصر الصفحة وتعديلها.
- الclient يلاقي في الhtml كود css وال js ف ال client بيبعت طلبات تانيه لتحميل باقي الملفات الcss بيتحمل و يتحول لCSSOM و دا و المتصفح غالبًا يؤخر رسم الصفحة لحد ما يخلص تحميله بنحط Js آخر body لأن تحميل وتنفيذ بعض ملفات Js ممكن يوقف تحليل HTML مؤقتًا.
- الbrowser بيدمج الDOM & CSSOM و بيعمل Render tree و بعدين يحسب مكان كل عنصر و بيرسم كل حاجه علي الشاشهو ده اللي بيشوفه المستخدم في الاخر
